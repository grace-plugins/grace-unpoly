      • Method Detail

        • boolean asBoolean()

        • java.lang.String getFailMode()

          This request header contains the mode of the layer targeted for a failed fragment update. A fragment update is considered failed if the server responds with a status code other than 2xx, but still renders HTML.

        • java.lang.String getFailTarget()

          This request header contains the target selector for a failed fragment update. A fragment update is considered failed if the server responds with a status code other than 2xx, but still renders HTML.

        • java.lang.String getHeaderValue(java.lang.String name)

        • java.lang.String getLocation()

          The server may set this response header to set a custom browser location after a fragment update. Without an X-Up-Location header Unpoly will set the browser location to the requested URL.

        • java.lang.String getMethod()

          The server may set this optional response header to change the HTTP method after a fragment update. Without this header Unpoly will use the requested method.

        • java.lang.String getMode()

          This request header contains the targeted layer's mode. Server-side code is free to render different HTML for different modes.

        • java.lang.String getTarget()

          This request header contains the target selector for a successful fragment update.

        • java.lang.String getTitle()

          The server may set this optional response header to change the document title after a fragment update. The title must be encoded as a JSON string.

        • java.lang.String getValidate()

          This request header contains the names of the form fields being validated.

        • java.lang.String getVersion()

          This request header contains the current Unpoly version to mark this request as a fragment update.
